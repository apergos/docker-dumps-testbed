What is this?

You have reached the main page of the SQL/XML Dumps testbed. This server is a mediawiki app/api container running apache. It communicates with a php-fpm instance that processes all requests except those for static content.

While this container listens on port 80, and should be available from the laptop or desktop running these containers, a separate web server may also be running and serving dump output files on 8080, if you have so configured the testbed.

You should be able to ssh into any container from your laptop or desktop, giving the fqdn of the container. The root password is "testing", and we plan to make that configurable in a future release.

See the document Testbed Containers describing the various containers and what they do, or just docker ls when they are running; the names should be pretty self-explanatory.

To check that the php-fpm container(s) are running properly, you may visit .

This testbed uses the PHP Opcache. If you make live changes to the version of MediaWiki being run in these containers, you'll need to invalidate the opcache without causing the container to exit. You can do that by visiting the Opcache invalidation page.

Please send bug reports, comments and suggestions to ops-dumps@wikimedia.org. Enjoy!
